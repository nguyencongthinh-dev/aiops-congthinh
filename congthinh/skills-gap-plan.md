# Bonus Section D: Skills-Gap & Transition Plan

Moving from a fully managed SaaS stack (Datadog, Splunk) to a self-hosted OSS stack (VictoriaMetrics, Loki, Tempo, OpenTelemetry) shifts the financial cost into an operational cost. This requires a targeted upskilling plan for three distinct personas within the engineering organization.

## 1. Platform Engineering & SRE Team
**The Gap:** Currently, the team only manages SaaS API keys and alert rules. They must now learn to provision, operate, and scale stateful OSS observability databases (VictoriaMetrics, Loki, Tempo), manage a clustered Redpanda queue buffer, configure Vector routing agents, deploy eBPF-based Grafana Beyla probes, and write automated remediation playbooks in Keep.
- **Strategy (Hiring vs Upskilling):** **Upskilling + Targeted Hiring.** We will upskill the existing team of 3 engineers. However, we must hire 1 Senior SRE with prior experience running high-throughput Loki/Prometheus clusters and message queues (Kafka/Redpanda) in production to act as the subject matter expert (SME).
- **Training Materials:** PromLabs PromQL training, Grafana Labs OSS administration courses, Redpanda University, Vector and Keep documentation.
- **Timeline:** 4 weeks prior to the "Shadow Mode" phase of the migration.
- **Cost:** ~$3,000 for specialized courses + ~$160,000/year for the new Senior SRE headcount.

## 2. Product Developers (SWE)
**The Gap:** Developers currently rely on Datadog's proprietary auto-instrumentation and custom query language. They must transition to understanding how Grafana Beyla performs zero-code auto-instrumentation via kernel-level eBPF, when to write custom application spans using OpenTelemetry SDKs, and how to use Grafana's query languages (PromQL for metrics, LogQL for logs, TraceQL for traces).
- **Strategy (Hiring vs Upskilling):** **Pure Upskilling.** No hiring required.
- **Training Materials:** Internal 2-hour workshops run by the Platform team. Creation of an internal "Grafana Cheat Sheet" mapping old Datadog queries to the new PromQL/LogQL equivalents.
- **Timeline:** Continuous over the 8-week migration.
- **Cost:** $0 in external materials; internal cost is ~4 hours of lost developer productivity per engineer during training.

## 3. Security & Audit Team
**The Gap:** The team relies heavily on Splunk Processing Language (SPL) to build compliance reports. They must pivot to using AWS Athena (Standard SQL).
- **Strategy (Hiring vs Upskilling):** **Pure Upskilling.** Security engineers already possess strong analytical skills. SQL is universally understood and easier to learn than SPL.
- **Training Materials:** AWS Athena documentation and a dedicated "SPL to SQL" translation guide provided by the Platform team.
- **Timeline:** 1-week intensive pairing session during Week 6 of the migration.
- **Cost:** $0 external.

## Total Investment Summary
The transition will cost approximately $3,000 in immediate training expenses and one additional Senior SRE headcount. This is heavily offset by the $32,000+/month ($384,000/year) savings generated by abandoning the SaaS vendors, making the ROI of this skills transition highly favorable.
